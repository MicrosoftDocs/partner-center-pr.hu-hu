- Képességek

#### <a name="software-in-mccl-csp-general-availability-date"></a>Az MCCL CSP-hez elérhető szoftverek általános elérhetőségének dátuma 

Az MCCL CSP programban regisztrált partnerek 2021 augusztusának első hetére megcélzva képesek lesznek folyamatos szoftverlicenceket értékesíteni a helyszíni telepítéshez az MCCL CSP-ben. Ez lehetővé teszi az ügyfelek számára a digitális átalakítás részeként szükséges licencelési rugalmasságot.

#### <a name="open-license-program-retirement"></a>Open License program kieső

2022. január 1-től az ügyfelek nem vásárolhatnak új vagy megújítható szoftverlicenceket a Microsoft Open License programon keresztül. Eddig az időpontig az ügyfelek továbbra is vásárolhatnak az Open License programon keresztül.

#### <a name="open-license-program-price-change-for-government"></a>Az Open License Program kormányzati árának módosítása

2021 júliusa óta a kormányzati ügyfelek különböző árnövekedést láthatnak az Open License program helyszíni termékkategóriáiban, például a Licenc (L), az Frissítési Garancia (SA) és a Microsoft License and Frissítési Garancia (L&SA) esetében. Ez a 2018-ban elindított munka folytatása, amely a kormányzati szervezetek árbeállítását az új kereskedelmi élményhez igazítja. Az Open License Government ára ekkor megegyezik a legalacsonyabb kereskedelmi árral, paritást hozva létre a CSP-program új kereskedelmi élményének díjszabásával, és megkönnyíti a kormányzati ügyfelek számára a megoldások összehasonlítását. További információt az Open License Government díjszabásával kapcsolatos [gyakori](https://aka.ms/openlicensegovpricefaq) kérdések [és az L100-csomag című cikkben talál.](https://aka.ms/openlicensegovpriceL100)

#### <a name="sa-purchases"></a>SA-vásárlások

Az SA előnyei addig maradnak, amíg az SA-időszak le nem jár, még akkor is, ha a lejárat 2022. január 1.. után van. A Microsoft Open Value (OV) program az SA-t is magában foglaló jövőbeli vásárlások esetén ajánlott.

>[!NOTE]
>Az OV és az Open Value Subscription programok módosítása nem tervben van.

## <a name="readiness-changes-to-the-cloud-solution-provider-csp-customer-address-validation-api-to-go-live-on-july-13-2021"></a><a name="8"></a>Készenlét: A Felhőszolgáltató (CSP) ügyfélcím-érvényesítési API-jának változásai, amelyek 2021. július 13-án indulnak el

### <a name="summary"></a>Összefoglalás

A Validate Address API módosításai 2021. július 13-tól lesznek elérhetőek.

### <a name="categories"></a>Kategóriák

- Dátum: 2021-07-09
- Készenlét

### <a name="impacted-audience"></a>Érintett célközönség

Felhőszolgáltató (CSP) közvetlen számlázási partnerek és közvetett szolgáltatók, akik új ügyfeleket hoznak létre vagy frissítik a címadatokat

### <a name="details"></a>Részletek

A Microsoft megbízhatósági kapcsolaton fut. Elkötelezettek vagyunk amellett, hogy megfelelő, biztonságos és biztonságos módszert biztosítsunk az ügyfélcímek érvényesítéséhez az ügyfél-előfizetések tranzakciója során a CSP-programban. 2021. július 13-án a Validate Address API módosításai élőek lesznek.
 
Vegye figyelembe, hogy ezek a módosítások csak a Validate Address API-t érintik. Az Ügyfél létrehozása és a Számlázási profil frissítése API-kat ez nem érinti. Jelenleg a javasolt címet nem kell használni az Ügyfél létrehozása API-val, de erősen ajánlott.

A válasz a következő állapotüzenetek egyikét adja vissza:

| Állapot | Leírás | A visszaadott javasolt címek száma |
|------------|---------------|------------------|
| Ellenőrzött szállításra használható  | A cím ellenőrizve van, és szállítható a címre.  | Egyirányú  |
| Ellenőrzött  | A cím ellenőrizve van.  | Egyirányú  |
| Beavatkozás szükséges  | A javasolt cím(ek) jelentősen módosultak/módosultak, és a felhasználó(k) megerősítésére van szükség.  | Egyirányú  |
| Utca részleges  | A címben megadott utca részleges, és több információra van szüksége.  | Multiple – legfeljebb 3  |
| Részleges helyszín  | Az adott helyszín (épületszám, csomagszám stb.) részleges, és további információkra van szüksége.  | Multiple – legfeljebb 3  |
| Többszörös  | Több mező is részleges a címben (lehetséges, hogy a Street partial és a Premises partial mezők is részlegesek).  | Multiple – legfeljebb 3  |
| None  | A cím helytelen.  | None  |
| Nincs ellenőrizve.  | A címet nem sikerült elküldeni az érvényesítési folyamaton keresztül.   | None  |

Miután elküldött egy címet a Validate Address API-val történő érvényesítésre, a rendszer a következő válaszsémát fogja visszaadni:

```csharp

// <summary>
/// Object represents the address validation response.
/// </summary>

public class AddressValidationResponse
{
   /// <summary>
   /// Gets or sets the original address
   /// </summary>
   /// <value>
   /// Original Address
   /// </value>
   public Address OriginalAddress { get; set; }

   /// <summary>
   /// Gets or sets the suggested addresses
   /// </summary>
   /// <value>
   /// Suggested Addresses
   /// </value>
   public List<Address> SuggestedAddresses { get; set; }

   /// <summary>
   /// Gets or sets the validation status
   /// </summary>
   /// <value>
   /// Status
   /// </value>
   public string Status { get; set; }

   /// <summary>
   /// Gets or sets the validation message
   /// </summary>
   /// <value>
   /// Validation Message
   /// </value>
   public string ValidationMessage { get; set; }

```

Vegye figyelembe, hogy Egyesült Államok válasz egy további négyjegyű utótagot ad vissza az irányítószám sorhoz, ha csak öt számjegyet ad meg az irányítószámhoz.
Az alábbiakban egy mintaválaszt követünk:

```C#

],

"country": "US", 
"city": "Redmond", 
"addressLine1": "1 Micro Way", 
"postalCode": "98052" 
}, 
"status": "InteractionRequired", 
"validationMessage": "Address field invalid for property: ‘Street’" 
}
```

## <a name="keep-your-microsoft-certifications-active-with-help-from-partner-center"></a><a name="6"></a>A Microsoft-tanúsítványok aktívak maradnak a microsoftos Partnerközpont

### <a name="categories"></a>Kategóriák

- Dátum: 2021-07-09
- Üzleti növekedés

### <a name="impacted-audience"></a>Érintett célközönség

A Microsoft minősítési állapotát és megújítását kezelő partner-rendszergazdák és globális rendszergazdák

### <a name="details"></a>Részletek

A Microsoft-tanúsítványok aktívvá és aktívvá teve egyszerűen kezelhetők a microsoftos Partnerközpont.

A szervezet tanúsítványai mindig is megtekinthetők voltak a Partnerközpont-irányítópulton, de a lejárati dátumukat a Felhasználói készségek jelentése és az Partnerközpont Elemzések Képzési jelentés is megtekintheti. Ezek a frissítések irányítják a szervezet tanúsítási megújítási folyamatát. A cikkből megtudhatja, hogyan találhatja meg a jelentéskészítési frissítéseket, és hogy azok milyen előnyöket fognak kihozni.

A saját irányítópultjának Kompetenciák Partnerközpont találja meg a kompetenciajelentést. Amikor a Excel-be exportálja az adatokat, megjelenik a felhasználói készségekkel kapcsolatos jelentés, amely felsorolja azokat az elért eredményeket (például tanúsítványokat és vizsgákat), amelyek minősítik a kompetenciákra. A jelentés végén egy új, "Skill Expiration Date" (Készség lejárati dátuma) feliratú oszlopban találja meg az alkalmazottak szerepköralapú és speciális tanúsítványai lejárati dátumát.  

Az irányítópult Szervezeti készségek Partnerközpont szakaszában találja a képzési Partnerközpont Elemzések jelentést. Amikor exportálja a jelentés adatait a Excel, egy új Lejárati dátum oszlopot fog találni, amely a csapat tanúsítványai lejárati dátumát tartalmazza.

Ezek a frissítések segítenek a gyors cselekvésben, és emlékeztetik az alkalmazottakat a tanúsítványaik megújítására, hogy a hitelesítő adataik és az MPN-program állapota is aktív maradjon.

Hogyan működik az új tanúsítvány megújítási folyamata?  

- Ahelyett, hogy egy tesztközpontba látogatna, és fizet a minősítő vizsgákért, az alkalmazottak felkeresik a Microsoft Learn-t, és átadnak egy ingyenes minősítési megújítási értékelést, amely teszteli őket a legújabb technológiai frissítéseken.
- A tanúsítvány lejárati dátumának közeledtével e-mailes értesítéseket kapnak a Microsofttól, és a tanúsítvány megújításának értékelésére irányítják Microsoft Learn.
- Az online képzési útvonalak tanulási anyagokat tartalmaznak az értékelésre való felkészüléshez, amelyeket annyiszor próbálkoznak, a ameddig csak szükségük van.
- Az értékelés átadása egy évre meghosszabbítja a tanúsítvány lejáratát.
- Ez a folyamat minden évben megismétlődik, és gondoskodik a készségek aktuális állapotáról.

Az új éves megújítási folyamathoz való igazodás érdekében a Microsoft két évről egy évre módosítja a szerepköralapú és speciális tanúsítványok érvényességi időkeretét. 2021. június 30-ával kezdődően minden szerepköralapú és speciális minősítés egy évig érvényes lesz.

Vegye figyelembe, hogy a Microsoft alapjai és a régebbi régebbi tanúsítványok mentesülnek alóla, és az új folyamat nem érinti.
